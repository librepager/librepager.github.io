Librepager Design Milestones

This page is an outline of the steps necessary to make the LibrePager a reality. The purpose is to keep the overall goals of the project in sight, while documenting progress to date. It is a living document, always being extended and improved. Milestones will be described, sometimes in little detail at a high level. As the work progresses, details will be fleshed out, links to pages documenting the technology will be added and, with some satisfaction, completion dates will be filled in.

LibrePager (LP) is a concept for a handheld text-messaging device that will directly with other LP devices forming self organizing peer to peer mesh network to extend range. Rooftop repeaters, internet gateways and patching into the SMS text message network is envisions. The goal is a system of Librepagers for daily use supplementing or replacing smartphones for normal communication between persons.

LibrePager are Meshtastic Transceivers, with a few extensions to make them practical replacements for texting on a smart phone. Hopefully, LP technology will always be compatible with Meshtastic technology.

  1. Build Meshtastic Transceivers (MTs) using current Meshtastic technology. (completed xx/xx/xxxx) Currently, MT devices are bluetooth connected dongles providing their Android or Apple hosts with radio links. Many important funcitons such as display and address books are handled in the hosts.
    • obtain commercially available dev boards, such as Heltec LoRa 32 V2, and a pile of android phones and tablets to use as host.
    • load Meshtastic by flashing the dev boards with pre-compiled binary firmware. and with binary images compiled from source code.
    • load Meshtastic apps now available from Google Playstore onto the Android host device.
  2. Build a standalone Meshtastic Transceiver (in progess):
    • build touchscreen testbeds (TS_Testbed) (future link) which drive touchscreens with dev boards running Meshtastic software. (x/x/xx).
    • design IOT-Terminal project to demo a prototype User Interface (UI) on the TS_Testbeds in hardware and in a simulator.
    • Design and build a prototype standalone MT hardware using a touchscreen and a dev board(s) which support LORA and does not require separate FCC certification.
    • Design software for MT standalone prototype joining IOT-Terminal User Interface, Meshtastic RF code and replacing all functions now accompished in host Android and Apple hosts.
  3. Build Prototype Meshtastic Repeaters to extend range of MTs mesh networks. This technology is currently under development with Meshtastic. Use dev board(s) which support LORA and do not require separate FCC certification.
  4. Build Prototype Librepager Internet Gateways. This is the first place where LibrePager devices will depart from and extend Meshtastic technology.
    • Prototypes will use dev board(s) which support LORA and do not require separate FCC certification. They will likely be similar to the Prototype Meshtastic Repeaters above
    • An LP_Internet Gateway will be a Meshtastic repeater with extensions to allow MT devices with LP extensions to link into the Internet. Connection to host computers inside buildings may be by wifi or ethernet. Many applications are possible.
    • LP2LP- An early app will allow LP devices in a local cluster to communicate with an LP device attached to a different LP_Internet_Gateway in a remote cluster. An addressing scheme, perhaps where LP-Internet Gateways generate ip6 addresses and lease them to the LPs, will be necessary.
    • LP2SMS- An LibrePager to SMS Text message network patch is another target application of the LP_Internet_Gateway. Using the Signal communications app, a command line interface (CLI), a patch into the telephone network's SMS texting systems willbe created.
  5. Build a single board Standalone Meshtastic Tranceiver (SMT) suitable for Librepager Service. An optimized single board mcu will be designed for this handheld unit that must be suitable for daily carry.
    • Design and test PCBs for single board SMTs.
    • Design and run software on previous generation of prototype hardware.
    • Perform FCC Testing and Certification if necessary. Crowd funding may be used to pay for this service and to fund the first run of LibrePager devices for commercial sale.
    • With careful planning, the same optimize tranceiver board which would be FCC tested and certified for handheld SMT could also be used for a series of improved repeaters and internet gateways
